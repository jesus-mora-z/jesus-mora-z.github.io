Finanzas + Código
+ IA

Trabajo en Humphreys Clasificadora de Riesgo desde 2021. Diseñé y mantengo en producción 7 sistemas de automatización que hoy procesan datos reales de la CMF, Larraín Vial y el sistema bancario de Chile y Perú.

Mi camino al desarrollo empezó con Power Automate. Cuando llegué al techo de la versión gratuita —sin connectors premium, sin HTTP custom para integraciones serias, sin gateway on-premises— vi que escalar pedía licencias caras o cambiar de stack. Aprendí Python para construir lo mismo sin costo de licencia, y de ahí seguí: VBA cuando los datos vivían en Excel, VB.NET cuando el caso pedía aplicación de escritorio, integración con LLMs cuando el proceso pedía comprensión semántica. El stack lo dicta el problema, no al revés.

Mi stack es Python para automatización pesada (Selenium, SQLite, APIs de IA), Power BI para análisis y dashboards internos, y HTML/JS para reportes web que se actualizan solos. Todo construido con Visual Studio Code y Visual Studio.

Trabajo con IA en niveles distintos según el problema. Los bots Python (CMF y Larraín Vial) los aprendí construyendo: al principio fue iterar con IA, copiar, pegar, leer el error, volver a preguntar — y a través de ese proceso entendí qué hace cada bloque. Hoy mantengo esos bots en producción, los modifico cuando un selector cambia o una API falla, y entiendo el flujo completo. Cuando estuvieron estables, usé IA agéntica en VS Code para modularizarlos y limpiarlos. Los reportes Power BI los hice 100% solo, sin IA. En los dashboards HTML dirijo a la IA: yo decido qué columnas y gráficos van, cuándo scroll, dark/light, qué extraer, qué cambiar; la IA escribe el código y yo itero hasta que el dashboard sirve a un analista. La diferencia importa: en una entrevista puedo decir exactamente qué hice yo en cada caso.

Lo que me diferencia es entender el dominio financiero y saber automatizarlo. Sé qué significa un documento CMF, qué indicador bancario importa, y cómo construir el sistema que lo procesa sin intervención humana.

→ Bot Larraín Vial: extrae alzas/bajas en tiempo real, captura gráficos y genera email HTML automático
→ 5 dashboards HTML con pipeline de actualización automática · CMF, BCRP, Humphreys

Bloomberg Rating Feed — Generador VBA

Macro VBA que consume la planilla diaria de clasificaciones de Humphreys — donde cada fila trae un comentario de Excel con texto crudo (Clasificación, Fecha, Tipo) — y genera el feed con el schema oficial de Bloomberg (23 columnas: Internal Security ID, Public Identifier, Public ID Type, Rating Type, Rating Effective Date, Country Code, etc.) listo para subir a la plataforma.

Mapeo dinámico de columnas por header → parser de comentarios de Excel (Clasificación, Fecha, Tipo) → jerarquía de Public Identifier (Nemo / NroInscripción / RunFondo) → clasificación de Rating Type (ACCIONES, CORTO PLAZO, CUOTAS FI/FM, RIESGO MERCADO M1-M6, SECURITIZADOS, LARGO PLAZO) → traducciones (NIVEL→PCN/SC, CONF→CONFIRMACION) → feed Bloomberg
Cómo lo construí: arranqué pidiendo a la IA un MVP minúsculo — "extrae los comentarios de la columna" — y fui agregando reglas una a una a lo largo de varios días: parser, jerarquía de identificadores, clasificaciones duales, sufijo M, diccionario de 30+ tickers de acciones (Concha y Toro → CONCHATORO, Watt's → WATTS, Multi X → MULTI X), diccionario de 60+ fondos (CFICELRI2, 9827, 10626…), reglas de Rating Type, traducciones de niveles. La IA escribió la sintaxis VBA; el dominio, la arquitectura y los 90+ mappings los aporté yo — son cosas que solo conoce alguien que vive en esta data.

Agente de Análisis Financiero — App de Escritorio

Aplicación Windows Forms en VB.NET compilada con Visual Studio. Acepta PDF/DOCX (Estados Financieros, Líneas de Bonos, Efectos de Comercio), extrae el texto con UglyToad.PdfPig (con freno inteligente a 15 páginas y filtro sanitario contra caracteres corruptos), y lo procesa con la API de DeepSeek usando prompts especializados por tipo de documento. Cada análisis abre en su propia pestaña ("Caso N"), de modo que el analista puede tener varios documentos en paralelo, y el resultado se copia automáticamente al portapapeles para pegarlo en el acta o el informe.

Construido a mano — la IA contestó dudas puntuales mientras programaba; el código (TabControl multi-caso, parsing de DOCX por ZipArchive, llamadas async con HttpClient, prompts por tipo de documento) lo escribí, integré y depuré yo en Visual Studio.

El pipeline del
Bot Hechos Esenciales

01 — EXTRACT
Monitoreo CMF
Selenium navega la sección de Hechos Esenciales de la CMF, filtra publicaciones nuevas según turno horario (mañana/mediodía/tarde/lunes) y descarga los PDFs de clientes Humphreys.
Selenium + Chrome
02 — FILTER
Historial & Dedup
Compara contra historial local para evitar procesar duplicados. Los lunes cubre desde el viernes 12:00. Normalización de nombres con super_normalize().
TXT historial + utils.py
03 — ANALYZE
IA: Gemini/DeepSeek
PDFs enviados a Gemini 2.5 Flash con rotación de 5 API keys. Si cualquier clave falla, fallback automático a DeepSeek. Extrae resumen relevante del documento.
Gemini + DeepSeek API
04 — DELIVER
Email + OneDrive
Genera email HTML con tabla de resultados y análisis IA. Adjunta PDFs y envía por Outlook a todo el equipo analista. Copia en OneDrive compartido.
pywin32 + Outlook
Impacto real: Procesa LATAM, Codelco, Entel, Concha y Toro, Enel, Colmena, Tanner y decenas de empresas más. Lo que antes requería revisar manualmente la CMF varias veces al día, hoy llega como resumen de IA al email del equipo. En empresas grandes esto se implementa con AWS Lambda + Bedrock o Azure Functions. Acá corre con Python en Windows — mismo resultado, distinto presupuesto.
Decisiones de ingeniería

Lo que aprendí
construyendo

Cosas que ningún tutorial enseña — solo aparecen cuando algo te falla en producción y tienes que arreglarlo a las 9 de la mañana antes de que alguien lo note.

01 · Rate limiting
Espaciar, no inundar
Aprendí que mandar todos los PDFs de una al LLM rompe la API. Ahora controlo tiempos entre requests para no quemar la cuota.
02 · Token optimization
Unificar PDFs antes de enviar
Un solo request con contexto unificado pesa menos y rinde más que N requests fragmentados. Decisión sencilla, ahorro real.
03 · Rotación de keys
5 cuentas free → un sistema
Diseñé un esquema con 5 API keys gratuitas que se turnan automáticamente respetando los límites de cada una. Escalar sin pagar.
04 · Fallback chains
Gemini cae → DeepSeek entra
Si Gemini falla por cuota, contenido o error de red, DeepSeek toma el relevo sin intervención. Producción no se detiene.
05 · Selectores estables
Recopilar de la web sin romperse
Scraping de la CMF, SBS, BCRP y Larraín Vial. Aprendí qué selectores resisten cambios de UI y dónde poner waits explícitos.
06 · Iteración guiada
Cada error me enseñó algo
Construir paso a paso, pegar un bloque, ver qué se rompía, leer el stack trace y corregir. Cinco años así te dejan entendiendo el flujo completo.
